# Claude Session Taskbar

A native macOS menu bar app that tracks your Claude Code session and weekly token usage in real time.

![macOS](https://img.shields.io/badge/macOS-13%2B-blue) ![Swift](https://img.shields.io/badge/Swift-5-orange)

## What It Does

- **Menu bar progress indicator** — a small coral-colored bar fills up as you use tokens
- **Click to expand** — shows detailed session and weekly usage with Claude-branded UI
- **Auto-refreshes** every 10 seconds by reading Claude Code's session JSONL files
- **Color shifts** from coral → amber → red as usage approaches your configured limits

## Quick Start

```bash
git clone <this-repo>
cd "Claude Session Taskbar"
chmod +x build.sh
./build.sh
open ClaudeSessionTaskbar.app
```

### Install permanently (optional)

```bash
./build.sh --install
```

This copies the app to `~/Applications/` and adds it to your Login Items so it starts on boot.

## Requirements

- macOS 13+ (Ventura or later)
- Swift toolchain (Xcode Command Line Tools: `xcode-select --install`)
- Claude Code installed and used at least once (the app reads from `~/.claude/projects/`)

## Configuration

On first launch, a config file is created at `~/.claude-taskbar.json`:

```json
{
  "refreshIntervalSeconds": 10,
  "sessionTokenLimit": 500000,
  "weeklyTokenLimit": 5000000
}
```

| Setting | Description | Default |
|---|---|---|
| `sessionTokenLimit` | Token budget for a single session (all token types combined) | 500,000 |
| `weeklyTokenLimit` | Token budget for a rolling 7-day window | 5,000,000 |
| `refreshIntervalSeconds` | How often the app re-reads session data | 10 |

Edit the file while the app is running — changes are picked up on the next refresh cycle. You can also click the gear icon in the dropdown to open the config file.

## How It Works

Claude Code stores conversation data as JSONL files in `~/.claude/projects/<project-hash>/`. Each API response includes a `usage` block with token counts:

- **Input tokens** — prompt tokens sent to the API
- **Output tokens** — tokens generated by Claude
- **Cache creation tokens** — tokens written to prompt cache
- **Cache read tokens** — tokens served from cache

The app scans all project directories, identifies the most recently active session (by file modification time), and sums token usage. Weekly data covers all sessions from the last 7 days.

## Quit / Restart

- **Quit**: Click the menu bar item → click the power icon
- **Restart**: `open ~/Applications/ClaudeSessionTaskbar.app` (or from the project: `open ClaudeSessionTaskbar.app`)

## Troubleshooting

**"No data" in the dropdown**
You haven't used Claude Code yet, or the `~/.claude/projects/` directory doesn't exist. Start a Claude Code session first.

**Build fails with SDK mismatch**
The build script auto-selects a compatible SDK. If it still fails, install Xcode Command Line Tools: `xcode-select --install`

**Progress bar shows 0% but I've used Claude**
Check that `~/.claude/projects/` contains `.jsonl` files. The app only reads completed API responses (lines containing `output_tokens`).
